# Efficient Algorithms and Intractable Problems

## What is an Algorithm?

An algorithm is a well-defined computational procedure that takes a set of values as **input** and produces a set of values as **output**.

- **Correctness**: An algorithm must **always halt** and produce the **correct** output for every possible input instance.
- **Efficiency**: We aim to minimize computational resources, specifically:
  - **Time Complexity**: How the running time grows as the input size $n$ increases.
  - **Space Complexity**: The amount of memory required during execution.
- **Asymptotic Analysis**: We use Big-O notation ($O$) to focus on the growth rate rather than constant factors or hardware-specific speeds.

------

## Addition vs. Multiplication

Before diving into Karatsuba, we analyze basic arithmetic:

- **Addition ($x + y$)**: Adding two $n$-bit (or $n$-digit) numbers takes **$O(n)$** time because we perform a constant amount of work for each digit (sum and carry).
- **Standard Multiplication**: The "grade-school" method requires multiplying every digit of $x$ by every digit of $y$, resulting in **$O(n^2)$** operations. Karatsuba’s algorithm was the first to prove we can do better.

------

## Divide & Conquer: Karatsuba's Algorithm

### 1. The Mathematical Derivation (Splitting the Numbers)

To multiply two $n$-digit numbers $x$ and $y$, we split them into two halves ($n/2$ digits each):

- $x = x_L \cdot 10^{n/2} + x_R$
- $y = y_L \cdot 10^{n/2} + y_R$

*(Example: If $x = 1234$, then $x_L = 12$ and $x_R = 34$. Thus $12 \cdot 10^2 + 34 = 1234$)*

The product $x \cdot y$ can be expanded as:

$$x \cdot y = (x_L \cdot 10^{n/2} + x_R)(y_L \cdot 10^{n/2} + y_R)$$

$$x \cdot y = \mathbf{x_L y_L} \cdot 10^n + (\mathbf{x_L y_R + x_R y_L}) \cdot 10^{n/2} + \mathbf{x_R y_R}$$

### 2. The Optimization (Reducing 4 Multiplications to 3)

A naive recursive approach requires 4 multiplications ($x_L y_L, x_L y_R, x_R y_L, x_R y_R$). However, Karatsuba discovered we only need 3:

1. Calculate $A = x_L \cdot y_L$
2. Calculate $B = x_R \cdot y_R$
3. Calculate $C = (x_L + x_R) \cdot (y_L + y_R)$

The middle term $(\mathbf{x_L y_R + x_R y_L})$ can be found by:

$$C - A - B = (x_L y_L + x_L y_R + x_R y_L + x_R y_R) - x_L y_L - x_R y_R = x_L y_R + x_R y_L$$

By using extra additions/subtractions (which are cheap, $O(n)$), we eliminated one expensive recursive multiplication.

### 3. Complexity Analysis

The recurrence relation for Karatsuba is:

$$T(n) = 3T(n/2) + O(n)$$

According to the **Master Theorem**:

- $a = 3$ (number of subproblems)
- $b = 2$ (factor by which input size is reduced)
- $d = 1$ (cost of addition/subtraction)

Since $\log_b a = \log_2 3 \approx 1.58 > d$, the complexity is:

**$T(n) = O(n^{\log_2 3}) \approx O(n^{1.58})$**

## Real-world Application: Python's `int`

While $O(n^2)$ is fine for small numbers, it becomes a bottleneck for large integers used in cryptography (like RSA) or scientific computing.

- **Hybrid Approach**: Python (CPython) uses a threshold for integer multiplication.
- **Threshold**: For small integers, it uses the standard $O(n^2)$ grade-school algorithm because the overhead of recursion in Karatsuba isn't worth it.
- **Switching**: When an integer exceeds a certain size (typically around **200–300 digits**), Python's internal C code automatically switches to **Karatsuba's Algorithm** to maintain high performance.
